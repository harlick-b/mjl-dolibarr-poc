# Grill With Docs

## When to use

Use this when product intent, business rules, terminology, role behavior, or
scope is ambiguous.

## Read first

- `AGENTS.md`
- `CONTEXT.md`
- Relevant docs under `docs/`

## Workflow

1. Extract confirmed facts from project memory and docs.
2. Separate confirmed behavior from needs-confirmation items.
3. Ask only questions that cannot be answered from the repo.
4. Convert the resolved intent into implementation-ready acceptance criteria.

## Output

Return confirmed facts, open ambiguities, recommended default, and acceptance
criteria.

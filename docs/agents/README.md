# Agent Configuration

MJL product decisions come from `../mjl-authoritative-decisions.md`.

This directory contains lightweight configuration consumed by global Matt
engineering skills. `AGENTS.md` remains the canonical routing and safety layer.

- `issue-tracker.md`: where PRDs, issues, and review tickets should be read or
  published.
- `triage-labels.md`: the tracker labels Matt planning and triage skills should
  use.
- `domain.md`: how skills should consume `CONTEXT.md`, `DESIGN.md`, and ADRs.

Keep detailed project facts in `CONTEXT.md`, detailed UI facts in `DESIGN.md`,
and reusable local workflow instructions in `skills/<name>/SKILL.md`.

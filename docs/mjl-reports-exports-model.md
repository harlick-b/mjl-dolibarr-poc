# MJL Reports And Exports Model

Target decisions come from `docs/mjl-authoritative-decisions.md`.

Status: `PENDING_CLIENT_VALIDATION`.

This document records the Phase 11R report/export model for UAT. It is not the
final donor/client canevas approval.

## Output Rules

- Formats: CSV and XLSX only.
- No PDF or Word reports in this phase.
- CSV is UTF-8 BOM, semicolon-separated, French-labeled, and Excel-readable.
- Exports are server-filtered and generated by POST with Dolibarr token
  protection.
- Filenames are stable and include practical filter context.
- Export generation creates `export_generated` audit rows.
- Report rows and options enforce active entity and Partenaire / Programme
  scope.
- Generic report-export audit rows remain Admin-only in scoped audit views
  until a first-class report-scope model is approved.

## Client-Readable Columns

Activity tracking should include Partenaire / Programme, project, activity
reference, title, responsible user, workflow status, execution status, planned
dates, actual dates, physical execution rate, associated budget, validated
amount, disbursed amount, and last decision.

Expense/disbursement tracking should include Partenaire / Programme, project,
activity, budget, expense reference, description, requested amount,
prevalidated amount, final-validated amount, disbursed amount, status, expense
date, disbursement date, justificatif presence, actors, and last decision.

Financial execution should include Partenaire / Programme, project, allocated
budget, submitted amount, prevalidated amount, final-validated amount,
disbursed amount, remaining balance, validation rate, and execution rate.

Physical execution should include Partenaire / Programme, project, planned
activities, activities in progress, executed activities, late activities, and
average physical execution rate.

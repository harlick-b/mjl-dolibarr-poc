# MJL Client Change Requests

This file is prepared for real client feedback. No change request is recorded
until a real client validation session provides it.

## Change request statuses

- NEW
- ACCEPTED
- REJECTED
- DEFERRED
- NEEDS_ESTIMATION
- IMPLEMENTED
- BLOCKED

## Priority levels

- BLOCKER
- HIGH
- MEDIUM
- LOW
- NICE_TO_HAVE

## Classification rules

- `BLOCKER`: prevents client validation or legal/business acceptability.
- `HIGH`: required before production release but does not block validation.
- `MEDIUM`: important improvement after validation.
- `LOW`: minor improvement.
- `NICE_TO_HAVE`: future enhancement.

Out-of-scope requests stay `DEFERRED` unless the user explicitly approves
expanding the scope.

Likely deferred examples:

- PDF/Word reports.
- SMS.
- OCR.
- Bank API.
- Public partner portal.
- Offline mode.
- Advanced BI dashboards.
- Automatic email notifications beyond configured app emails.

# MJL Clarity System — Official Outputs

## Purpose

Define how reports, exports, PDFs, and print views should look and behave.

## Baseline Constraints

- Dolibarr core files must not be modified.
- MJL-specific implementation must remain inside safe custom module/theme boundaries.
- The temporary access model is exactly Level 1, Level 2, Level 3, Admin.
- Access is invitation-only.
- Only Admin can send invitations for now.
- There is no public register page.
- The design system covers app UI, auth pages, system emails, official outputs, and E2E tests.
- E2E tests are the main validation method.

## Export Principles

Exports must be predictable, role-aware, Excel-readable, stable in filename, clear about filters, and consistent with French labels.

## Export UI

Each export page should show report name, description, filters, period, format, scope, and generated filename preview if possible.

Example:

```txt
Exporter les activités filtrées
Période : 01/01/2026 - 31/12/2026
Format : Excel compatible
```

## Report Layout

If PDF or printable reports are added, include title, project/activity reference, period, generated date, generated by, status, decision history if relevant, signature/validation block if needed, and footer.

## French Formats

Use:

```txt
26/06/2026
1 250 000 FCFA
```

Avoid ambiguous dates such as `06/26/2026`.

## Stable Filenames

Examples:

```txt
mjl_activites_2026-01-01_2026-12-31.csv
mjl_alertes_2026-06-26.csv
mjl_audit_activite_A-2026-014.csv
```
